# Software Instances
Use the **Software Instances** page to manage myViewBoard installations that have been deployed and enrolled under your entity. The following tasks are available:
  

  * Review details for each myViewBoard installation
  * Adjust settings for each myViewBoard installation
  * Apply a settings template to multiple myViewBoard installations at once


  

This page displays all myViewBoard installations deployed on Windows, Android, and iOS environments, as long as they have been successfully enrolled under the entity.
  

  

  

## How to access
To access this page:
  1. Sign in to 
  2. On the landing page, locate the **myViewBoard** tile.   
Note the ellipsis icon when you hover over.
  3. Click the **myViewBoard admin settings**.
  4. The **myViewBoard admin settings** page appears.
  5. Click **Software Instances** on the side panel.


  

  

**Important**
  

  * Any changes made here only apply to your myViewBoard environment. 
  * You can only access this page if any of the following conditions are true: 
    * You have signed in under a myViewBoard owner account 
    * You have been assigned with the Admin role for the myViewBoard app

## Enroll myViewBoard instances 
On first use (or after Entity creation), the Software Instance page will be empty. To get your installations listed on the Software Instance page, required entity information must be explicitly provided during installation by specifying additional parameters or using specialized installers. 
  

For more information, refer to the following guides: 
  * [Deploy myViewBoard to Windows PCs](https://www.viewsonic.com/solution/kb/myviewboard-deployment)
  * [Deploy myViewBoard to Android / iOS](https://www.viewsonic.com/solution/kb/deploy-to-android-ios)


  

  

## All instances
All myViewBoard installations that have been successfully enrolled will appear in this list, along with the following information: 
  

Field | Description  
---|---  
Instance ID | An auto-generated unique ID assigned to this instance during installation. Click to change settings for this instance.  
myViewBoard Name | The myViewBoard name provided during installation. Used to verify, identify, or search for a specific myViewBoard instance.  
Last User | The email address of the most recent user who was able to successfully log in on this installation.  
OS | Displays icons that indicate the operating system where the instance is installed.  
Version | The current version of the instance installed. Used to check if a myViewBoard instance needs to be updated.  
IP Address | Last known IP address assigned to this instance.  
Template | Name of the template applied to this instance. Indicates if options for this instance are currently set by a template.  
  

  

To make changes to multiple instances, perform the following:
  

  1. Locate the instances you want to configure. Use one of the following tools:
     * _**Filter by OS**_. Show only the instances that are running on the selected OS.
     * _**Search**_. Search by Instance ID, myViewBoard Name or Version.  
  

  2. Select one or more _checkboxes_.
  3. Click one of the following:  
|  **Edit**  
Change settings on the selected instances.  
---|---  
|  **Move to**  
Add the selected instances to a group. Go to the Groups tab to create a new group.  
|  **Delete**  
Remove this instance from the list. This does not uninstall myViewBoard.  


  

  

## Groups
To apply similar commands to multiple instances, organize your instances into groups first. 
  

### Create new group
To create a new group, perform the following:
  

  1. Go to _**All instances**_.
  2. Go to **Groups** tab.
  3. Click **Create new group.**
  4. On the popup:
    1. Provide a group name.
    2. Click **Create new group**.  
  

  5. The new group is automatically loaded in the section below.


  

### Update group info
To update a group, perform the following:
  

  1. From the **Group** dropdown, select the group to update. 
  2. Wait for myViewBoard to load the group details.
  3. Use the following tools:  
|  **Edit group name**  
Changes the group name. Make sure that the new name you provide is unique.  
---|---  
|  **Delete**  
Removes the group from the list. This does not delete instances added as members.  


  

### Add group members
To add instances to a group, perform the following:
  

  1. From the **Group** dropdown, select the group to update. 
  2. Wait for myViewBoard to load the group details.
  3. Use the following tools:  
|  **Add instances**  
Displays a popup where you can select one or more instances to add to this group.  
---|---  


  

Once you have instances added, click the Instance ID of any entry you want to update.
  

### Update multiple
To make changes to multiple instances, perform the following:
  

  1. Locate the instances you want to configure. Use one of the following tools:
     * _**Filter by OS**_. Show only the instances that are running on the selected OS.
     * _**Search**_. Search by Instance ID, myViewBoard Name or Version.  
  

  2. Select one or more _checkboxes_.
  3. Click one of the following:  
|  **Edit**  
Change settings on the selected instances.  
---|---  
|  **Move to**  
Add the selected instances to a group.  
|  **Remove**  
Remove this instance from the group. This does not uninstall myViewBoard.  


  

  

## Templates
Use templates to create pre-defined configurations that can be readily applied to multiple instances. 
  

### Create new template
To create a new template, perform the following:
  

  1. Go to _**Templates**_.
  2. Click **Create new template.**
  3. On the popup:
    1. Provide a **Template name**.
    2. Configure settings for this template.
    3. Click **Save**.  
  

  4. The new template is automatically loaded in the section below.


  

### Update template info
To update a template, perform the following:
  

  1. From the **Template** dropdown, select an entry to update. 
  2. Wait for myViewBoard to load the template details.
  3. Use the following tools:  
|  **Default**  
Sets the commands in this template as the default settings to apply for new instances.  
---|---  
|  **Delete**  
Removes the template from the list. This does not delete instances that use the template.  
|  **Add instances**  
Displays a popup that allows you to apply this template to one or more instances.  
|  **Reset**  
Select one or more instances, then click to apply whatever template has been set as Default to the selected instances.  


  

  

### Template settings
Specify how users can interact with each instance. The following options are available:
  

  * **Template name**   
Specify a unique name for this template.   
  

  * **Check for Update**   
Allow instance to check for the latest update and prompt users to install it if available.  
  

  * **Local Save**   
Allow users to save files in a device's local storage.  
  

  * **Embedded Browser**   
Give users access to the **Embedded Browser** function. If turned **off** : 
    * The browser icon will not be available.  
  

  * **Sign-in Function**   
Allow users to sign in. If turned **off** :
    * Disables the sign-in prompt so users cannot sign in. 
    * Overrides the **Allow Stay Signed-in** and **Auto Sign-out Time** options below.  
  

  * **Allow Stay Signed In**   
Allow users to set whether they want to stay signed in to myViewBoard. Note that **Allow Stay Signed in** is only available in myViewBoard for Windows. If turned **on** :
    * Users remain signed in even after application closure or device shutdown. 
    * Overrides **Auto Sign-out Time**.  
  

  * **Auto Sign-out Time**   
Specify how long the myViewBoard instance should remain idle before it automatically signs out.


  

  

Note that any change made on the **Template settings** section are automatically saved.
