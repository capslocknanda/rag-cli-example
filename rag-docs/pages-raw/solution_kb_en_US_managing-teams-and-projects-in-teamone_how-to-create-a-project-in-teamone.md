# Create a project
After you've set up your team, create projects and assign which members will have access to its boards. Admins can also assign different access rights to each member, giving them either editing or viewing rights to specific boards within the project.
  

  

## Create a project
Note that only users with the **Team admin** role can create new projects. 
  

  1. Go to the TeamOne dashboard.
  2. On the left panel, select a team to handle the new project. 
    1. Click the team dropdown.
    2. Select a team where you have **Team admin** rights.   
  

  3. On the left panel, click **Project** label.
  4. On the new window, provide the following information:
    1. **Project name**. Specify a unique name for this project.
    2. **Project description**. Add details. Optional.
    3. **Project type**. This is set to **Private** by default.  
  

  5. Click **Create project**. 
  6. The project is added as a new entry on the left panel.


  

  

## Add project contributors
Afterwards, assign members who should become project contributors and specify their access rights.
  

  * If _Project type_ is **Private** , you'll be prompted to assign members to the new project.
  * If _Project type_ is **Public** , all members in the current team have access to the new project and view its boards.


  

To add project contributors, follow the steps below:
  

  

  1. Go to the TeamOne dashboard.
  2. Select the team that contains your target project.
  3. On the right panel, click **Share project** beside the project name.  
  

  4. To add members via email: 
    1. Type an email address in the input box under the '**By email** ' label.
    2. Press Space or Enter to add the email address as a new entry. 
    3. Type additional email addresses as needed. 
    4. Click 
    5. Once you've added all email addresses, click **Send invitation**. 
    6. TeamOne sends an email to all recipients with an invite link to the current project.  
  

  5. To add members via link:
    1. On the input box under the '**By link** ' label, click **Copy Link**. This will save the invite link to the system clipboard.
    2. On your chosen platform (email, sms, chat, etc.), add this link to messages that you will be sending to invite contributors to your project.
    3. To reset the invite link, click the   
  

  6. Once users click the invite link: 
    1. If they are already signed in to a TeamOne account, they will be automatically added as member.
    2. If they are not signed in to a TeamOne account, they will be asked to sign in first, then automatically added as member.
    3. If they do not have a TeamOne account, they will be asked to create one. They will be automatically added as members after they complete the sign up process.


  

  

## Manage project users
**Private** projects allow admins to update the roles of members even after these members have joined:
  

  1. Go to the TeamOne dashboard.
  2. Select the team that contains your target project.
  3. On the left panel, select a project which is **Private**.
  4. On the right panel, click **Contributors** beside the project name.  
  

  5. On the **Manage project users** window: 
    1. Locate the user to update.
    2. Click the role beside the target user and select a different value.
    3. Click **Done**.  
  

  6. If the project is **Public** , you need to set the _Project type_ to **Private** first before you can edit user roles:
    1. On the right panel, click **Settings** beside the project name
    2. Change the **Project type** to **Private**.
    3. Click **Save**.
    4. Perform the same steps described above.  
  

  7. TeamOne sends an email to all updated users to inform them of the change.
